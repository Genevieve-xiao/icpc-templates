## 数据范围推算法

时间限制是1秒或2秒时，C++代码中的操作次数控制在$10^7 \sim 10^8$。
在不同数据范围下，代码的时间复杂度和算法的选择：

1. $n \leq 30$，指数级别，dfs+剪枝，状态压缩dp
2. $n \leq 100 \Rightarrow O(n^{3})$，floyd，dp，高斯消元
3. $n \leq 1000 \Rightarrow O(n^{2})$，$O(n^{2}\log n)$，dp，二分，朴素版Dijkstra、朴素版Prim、Bellman-Ford
4. $n \leq 10000 \Rightarrow O(n * \sqrt{n})$，块状链表、分块、莫队
5. $n \leq 100000 \Rightarrow O(n\log n) \Rightarrow$ 各种sort，线段树、树状数组、set/map、heap、拓扑排序、dijkstra+heap、prim+heap、Kruskal、spfa、求凸包、求半平面交、二分、CDQ分治、整体二分、后缀数组、树链剖分、动态树
6. $n \leq 1000000 \Rightarrow O(n)$，以及常数较小的$O(n\log n)$算法 $\Rightarrow$ 单调队列、hash、双指针扫描、BFS、并查集，km p、AC自动机，常数比较小的$O(n\log n)$的做法：sort、树状数组、heap、dijkstra、spfa
7. $n \leq 10000000 \Rightarrow O(n)$，双指针扫描、km p、AC自动机、线性筛素数
8. $n \leq 10^{9} \Rightarrow O(\sqrt{n})$，判断质数
9. $n \leq 10^{18} \Rightarrow O(\log n)$，最大公约数，快速幂，数位DP
10. $n \leq 10^{1000} \Rightarrow O((\log n)^{2})$，高精度加减乘除
11. $n \leq 10^{100000} \Rightarrow O(\log k \times \log\log k)$，$k$表示位数，高精度加减、FFT/NTT

# 除法与取模
## 定义介绍
| 流派 | 取整方式 | 余数 $r$ 的符号 | 典型代表 |
| --- | --- | --- | --- |
| **Truncated Division** | 向零取整 (Trunc) | 与被除数 $a$ 一致 | **C, C++, Java** |
| **Floored Division** | 向下取整 (Floor) | 与除数 $b$ 一致 | **Python, Haskell** |
| **Euclidean Division** | 欧几里得除法 | **永远为正 (Non-negative)** | **数学定义** |

> （笔者不用python打算法比赛所以不讨论python）

在数学中，关于负数取模（或余数）的定义主要基于**带余除法**。

对于任何两个整数 $a$（被除数）和 $b$（除数，$b \neq 0$），一定存在唯一确定的两个整数 $q$（商）和 $r$（余数），使得：$a = b \cdot q + r$，且满足约束条件：$0 \le r < |b|$
因此，在纯数学定义中，**余数 $r$ 必须是非负的**。

**具体举例：`-5 % 3`**

数学方式：
>为了满足 $0 \le r < 3$：
>* 我们需要找到一个商 $q$，使得 $3 \cdot q \le -5$ 且最接近 $-5$。
>* 选取 $q = -2$。
>* 计算：$-5 = 3 \cdot (-2) + r \implies -5 = -6 + r \implies r = 1$。
>* **结果：** $-5 \pmod 3 = 1$。

 C++/Java（向零取整）方式：
>这些语言先做除法 `(-5 / 3)`，结果是 `-1.66...`，向零取整得到商 $q = -1$。
>* 计算：$-5 = 3 \cdot (-1) + r \implies -5 = -3 + r \implies r = -2$。
>* **结果：** `-5 % 3 = -2`。




## 极致的取模

### 大数取模

```cpp
ll add(ll a, ll b){
	return ((a % MOD + b % MOD) % MOD + MOD) % MOD;
}
ll multi(ll a, ll b){
	return (a % MOD) * (b % MOD) % MOD;
}
```

### 负数取模的正数化

C++/Java 的 `%` 运算符在处理负数时结果为负（如 `-5 % 3 = -2`），需要转化为数学中的取模。

```cpp
(a % mod + mod) % mod
```

## 开区间变闭区间

$x < k \implies x \le \lceil k \rceil - 1$
$x > k \implies x \ge \lfloor k \rfloor + 1$
在可能会涉及到浮点型的题目中可以用这种方法进行规避，并且简化了边界问题。

## 手写四舍五入

$$\lfloor \frac{a + b / 2}{b} \rfloor$$

在代码中也就是`(a + b / 2) / b` ，或者写成更严谨的 `(a * 2 + b) / (b * 2)`。
对应`round()`函数。

## 手写上取整
整型除法的上取整可以用这个公式。（当然也可以用 `ceil()` 函数）
$$\lceil \frac{a}{b} \rceil=\lfloor \frac{a+b-1}{b} \rfloor，(a,b>0)$$

代码中也就是`(a + b - 1) / b` ，但是当a+b超int会出错（除非你记得开longlong了），更稳健的写法是 `1 + (a - 1) / b` 

> 所以总而言之，当都是正数的时候，上取整用`1 + (a - 1) / b` ，下取整就用`/`（记得强制类型转化为double）。涉及到非正整数的时候，就用`ceil()` 和 `floor()` 吧，但是注意这两个函数返回的是double类型。

### 取整问题拓展

#### 1. 嵌套取整的“合并”性质

在处理分级分配或递归分治（如数位 DP 或分块）时：


$$\lfloor \frac{\lfloor a/b \rfloor}{c} \rfloor = \lfloor \frac{a}{bc} \rfloor$$

$$\lceil \frac{\lceil a/b \rceil}{c} \rceil = \lceil \frac{a}{bc} \rceil$$

> **应用：** 如果要把 $n$ 个元素分成 $b$ 组，再把这些组每 $c$ 个合成一个大组，那么大组的数量直接就是 $\lfloor n / (bc) \rfloor$。

#### 2. GCD（最大公约数）的差分性质

“区间加”同时询问“区间 GCD”的问题（如线段树维护 GCD）：


$$\gcd(a, b) = \gcd(a, b - a)$$


推广到序列：


$$\gcd(a_1, a_2, \dots, a_n) = \gcd(a_1, a_2 - a_1, a_3 - a_2, \dots, a_n - a_{n-1})$$

> **应用：** 当一个区间 $[L, R]$ 同时加上一个数 $v$ 时，序列的**差分数组**只有两端会变，中间部分不变。这样就可以用线段树轻松维护动态区间的 GCD。

#### 3. 区间内的倍数统计

统计区间 $[L, R]$ 中有多少个数能被 $k$ 整除：


$$\lfloor \frac{R}{k} \rfloor - \lfloor \frac{L-1}{k} \rfloor$$

> **注意：** 这个公式在 $L=0$ 时需要特判，或者根据题目定义确定 $0$ 是否算作倍数。

# 位运算

* **Lowbit (获取最低位的 1)**: `x & -x`（常用于树状数组）。
* **消除最低位的 1**: `x & (x - 1)`。
* *判定 $n$ 是否为 2 的幂次：* `n > 0 && (n & (n - 1)) == 0`。


* **二分搜索项数**: 如果要在 $[0, 2^k-1]$ 找东西，位运算通常能代替很多繁琐的边界判断。
* **判断奇偶性**：用 `(n & 1)` （负数适用）代替 `n % 2 == 1`（负数不适用）。

# 读写技巧
## iostream加速
```cpp
ios::sync_with_stdio(false);
cin.tie(nullptr);
```
加了之后会加速 `cin` `cout` ，但是就不要再和 `scanf` `printf` 混用了。

## 快读快写

```cpp
int read(){
	int x=0,f=1; char ch=getchar();
	if(ch=='-') f=-1, ch=getchar();
	while(ch<'0'||ch>'9') ch=getchar();
	while(ch>='0'&&ch<='9') x=(x<<1)+(x<<3)+(ch^48), ch=getchar();
	return x*f;
}
```

## EOF
对于不知道何时停止的输入
```cpp
while (cin >> x) {}
while (cin >> a >> b) {}
while (getline(cin, s)) {}
while (scanf("%d", &x) != EOF) {}
while (scanf("%d", &x) == 1) {}
while (scanf("%d%d", &a, &b) == 2) {}
```

## 字符串与数字混合读入

假如读入的一行为固定数目的单词+若干数字
```
odd 2 3
```
一般采用一个字符数组配合 `scanf` 来读入，`%s` 可以忽略一些莫名其妙的行末空格，而 `getchar()` 不行
```cpp
char op[5]; int a,b;
scanf("%s%d%d", op, &a, &b);
```

## 字符串整行读入一句英文句子

```cpp
string line;
getline(cin, line);
```
`getline(cin, line);` 的意思是：从标准输入 `cin` 里读取一整行内容，并把这一整行保存到字符串变量 `line` 里。
单独的 `cin` 会被空格分隔，如果直接用 `cin>>line;` 的话，会在第一个空格前停下。


## 交互题

每次输出询问后必须刷新输出缓冲区。
C++：
```cpp
cout << "? " << x << " " << y << endl;
```
endl 会自动换行并刷新缓冲区，所以可以用。

也可以写成：
```cpp
cout << "? " << x << " " << y << '\n';
cout.flush();
```

如果用 printf：
```cpp
printf("? %d %d\n", x, y);
fflush(stdout);
```

# 图论

## 建图：二维坐标转一维坐标
情景：将网格视为一张图，网格中的每个交点视为图中的点，但是初始网格中每个点以二维坐标的形式给出，然而图论中一般用一个数作为某个点的唯一标识。网格默认 $n$ 行 $m$ 列。

当x，y都从0开始，则可以：
```cpp
id = x * m + y
```
此时 id 也是从0开始，如果想让 id 从1开始的话，直接+1就好了。

当x，y都从1开始，则可以：
```cpp
id = (x - 1) * m + (y - 1)
```
此时 id 是从0开始，如果想让 id 从1开始的话，直接+1就好了。
